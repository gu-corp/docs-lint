# ビジネス要件

**バージョン**: 1.18.0
**更新日**: 2026-02-18

---

## 概要

docs-lint のビジネス要件を定義します。このツールの存在意義と達成すべきビジネス目標を明確にします。

## 背景

G.U.Corp では90以上のリポジトリを管理しており、ドキュメントの品質・構成がプロジェクトごとに異なる状況でした。これにより：

- 新メンバーのオンボーディングに時間がかかる
- ドキュメント間の整合性が取れない
- AIツールでのドキュメント活用が困難

## ビジネス要件一覧

| ID | 要件 | 優先度 | ステータス |
|----|------|--------|----------|
| BR-001 | G.U.Corpの90+リポジトリのドキュメント品質を標準化する | 高 | 達成 |
| BR-002 | AIによるドキュメントレビューを支援する | 中 | 達成 |
| BR-003 | CI/CDパイプラインでの自動チェックを可能にする | 高 | 達成 |

## 要件詳細

### BR-001: ドキュメント品質の標準化

**目的**: 全リポジトリで一貫したドキュメント構成・品質を実現する

**成功基準**:

- 標準フォルダ構成（01-plan, 02-spec, 03-guide, 04-development）の採用
- 命名規則の統一（UPPER-CASE.md）
- 必須セクション（バージョン情報、関連ドキュメント）の存在

**KPI**:

- 導入リポジトリ数
- リント通過率

### BR-002: AIドキュメントレビュー支援

**目的**: AIツール（Claude等）がドキュメント評価を効率的に行えるようにする

**成功基準**:

- `--ai-prompt` オプションで構造化されたプロンプトを生成
- ドキュメント標準をAIに提供可能
- 機械可読なJSON出力

**KPI**:

- AIレビューの精度向上率

### BR-003: CI/CD統合

**目的**: プルリクエスト時に自動でドキュメント品質をチェックする

**成功基準**:

- 終了コード0/1でCI判定可能
- GitHub Actionsでの動作確認
- エラー時の明確なメッセージ

**KPI**:

- CI導入リポジトリ数
- ドキュメント品質問題の早期検出率
