# 非機能要件

**バージョン**: 1.18.0
**更新日**: 2026-02-18

---

## 概要

docs-lint の非機能要件（性能、互換性、保守性、セキュリティ等）を定義します。

## 非機能要件一覧

### 性能要件

| ID | 要件 | 基準 | 測定方法 |
|----|------|------|----------|
| NFR-PERF-001 | 100ファイルのリントが10秒以内に完了する | 10秒 | ベンチマークテスト |
| NFR-PERF-002 | メモリ使用量が500MB以下 | 500MB | プロファイリング |

### 互換性要件

| ID | 要件 | 基準 | 備考 |
|----|------|------|------|
| NFR-COMPAT-001 | Node.js 18以上で動作する | Node 18+ | LTS対応 |
| NFR-COMPAT-002 | ESMモジュールとして提供する | ESM | import構文対応 |
| NFR-COMPAT-003 | Linux/macOS/Windowsで動作する | クロスプラットフォーム | CI確認 |

### 保守性要件

| ID | 要件 | 基準 | 備考 |
|----|------|------|------|
| NFR-MAINT-001 | 依存パッケージを最小限に保つ | 10個以下 | セキュリティリスク低減 |
| NFR-MAINT-002 | TypeScriptで型安全を確保 | strict: true | 型チェック必須 |
| NFR-MAINT-003 | テストカバレッジ70%以上 | 70% | コアロジック100% |

### 信頼性要件

| ID | 要件 | 基準 | 備考 |
|----|------|------|------|
| NFR-REL-001 | 不正な入力でクラッシュしない | - | エラーハンドリング |
| NFR-REL-002 | 明確なエラーメッセージを表示 | - | ユーザー理解可能 |

## 詳細

### NFR-PERF-001: 処理速度

**背景**: 大規模プロジェクトでもCIパイプラインの待ち時間を許容範囲内に収める

**測定方法**:

```bash
time npx docs-lint lint -d ./docs
```

**現状**: 100ファイルで約3秒（達成）

### NFR-COMPAT-001: Node.js互換性

**背景**: G.U.Corpの各プロジェクトはNode.js 18以上を使用

**確認方法**:

- GitHub ActionsでNode 18, 20, 22での動作確認
- package.jsonの`engines`フィールドで明示
